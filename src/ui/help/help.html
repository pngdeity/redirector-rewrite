Quick Start

Redirector intercepts outbound web requests matching your custom Include Pattern, excludes those matching your Exclude Pattern, and routes them to your Redirect Target URL.

To create your first redirect:

  1. Click the Redirector icon in your browser toolbar and select Edit Redirects.
  2. Click Create new redirect.
  3. Fill in the fields as described below. Use the Live Test section to preview matches in real-time.
  4. Click Save. Your redirect is now active.

Redirects are evaluated in order from top to bottom. Use the ▲/▼ arrows to reorder them by priority.

Pattern Types

1. Wildcards

Wildcards are the easiest way to match URLs. The asterisk character (*) matches any sequence of characters (non-greedy, meaning it matches the shortest possible string). You can use multiple stars in one pattern.

Wildcard Examples

PatternMatches
http://example.com/*http://example.com/, http://example.com/foo, http://example.com/bar
http://*.example.comhttp://www.example.com, http://mail.example.com
*://example.comhttp://example.com, https://example.com

Capture Groups with Wildcards

Each * in your pattern becomes a capture group. Use $1, $2, etc. in your redirect target to refer to them:

Include Pattern: *://*.wikipedia.org/wiki/*
Redirect Target: https://wikipedia.org/wiki/$3

$1 = protocol, $2 = subdomain, $3 = article title.

2. Regular Expressions

Regular expressions (regex) provide complete power over URL matching. Use parentheses (...) to create capture groups and refer to them as $1, $2, etc. in the redirect target.

Regex Examples

Include Pattern: ^https?:\/\/([^\.]+)\.slack\.com\/(.*)
Redirect Target: https://slack.com/workspace/$1/path/$2

Captures the Slack workspace name as $1 and the path suffix as $2.

Include Pattern: ^(https?://)(www.)?youtube.com/shorts/(.*)
Redirect Target: $1www.youtube.com/watch?v=$3

Redirects YouTube Shorts to regular YouTube. Check "HistoryState" in Apply To for SPA navigation.

Tip: Use a site like RegExr to test your regular expressions. Remember to escape special characters: ? → \?, . → \., / → \/.

Match Processing Options

When substituting captured groups into your target URL, you can transform them:

  • No Processing: Injects the raw captured string.
  • URL Decode: Decodes URI sequences (e.g. converting %20 back to spaces or %2F to /). Useful for redirecting tracking or routing parameters.
  • URL Encode: Safely encodes components (e.g. converting special characters to URI sequences).
  • Double URL Decode: Decodes twice — for URLs that have been encoded multiple times. Falls back to single decode if double decoding fails.
  • Base64 Decode: Decodes base64-encoded strings (useful for extracting encoded target URLs from landing redirects).

Apply To (Request Types)

By default, Redirector only intercepts main window navigation (address bar requests). You can expand this to other resource types:

  • Main Window: Normal page navigation in the address bar.
  • IFrames: Embedded frames within pages.
  • Scripts / Stylesheets / Images / Fonts: Resource requests of those types.
  • Ajax (Fetch/XHR): API calls and dynamic content loading.
  • HistoryState (SPAs): Intercepts pushState/replaceState navigation in single-page apps (e.g., Facebook, Twitter, YouTube). Requires the rule to NOT be sent to DNR — engine-only evaluation.

Performance note: Expanding to script/image/font types adds overhead. Only check types you actually need to redirect.

Storage Sync

Enable Storage Sync to share your redirect rules across devices via your browser's sync service (Chrome Sync or Firefox Sync).

Limits:

  • Local storage: 10 MB (default). Recommended for most users.
  • Sync storage: ~100 KB total, each item max 8 KB. Rules exceeding the per-item limit will be rejected.

If you have many rules or long patterns, keep sync disabled to avoid capacity errors.

Organize Mode & Grouping

Click Organize on the dashboard toolbar to enter organize mode:

  • Grouping checkmarks: Link adjacent rules by checking the box. Grouped rules:
    • Enable/disable together
    • Move as a block when reordering
  • Jump arrows (⟰/⟱): Move rules to the top or bottom of the list instantly.
  • Visual indication: Grouped rules share a blue left border with dashed bottom edges.

Groups are also editable inside the rule editor under Advanced Options.
